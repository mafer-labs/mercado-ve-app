# 🛒 CuentaClara (Calculadora de Mercado VE)
> *"Tu mercado al mejor cambio."*

CuentaClara is a Progressive Web App (PWA) designed specifically for the Venezuelan economic context. It helps users track grocery costs in USD, calculates the equivalent in Bolívares (VES) using the official BCV rate, and computes how many USDT (Binance) they need to sell to cover the bill. 

It also functions as a **crowdsourced price tracker**, building a community database of supermarket prices to help users find the cheapest locations.

---

## 🌟 Key Features

### 1. Smart Calculator & Draft Mode (Modo Planificación)
* **Plan at Home, Fill at the Store:** Users can create shopping lists with items, quantities, and units (und, kg, g, L) while leaving the price blank. The item will show as "Pendiente ⏳".
* **On-the-fly Editing:** Once at the supermarket, users click the edit icon (✏️) to input the real price.
* **Smart Units:** Automatically defaults to `kg` when selecting items from the "Produce" or "Meats" aisles. Accepts decimal quantities (e.g., 1.5 kg).

### 2. Dual Exchange Rate Engine
* Automatically fetches real-time official **BCV (Central Bank of Venezuela)** and **Binance USDT (P2P)** exchange rates via the *PyDolarVenezuela API*.
* Allows manual override of rates if the specific supermarket uses a different rate.
* Calculates the "Ahorro" (Savings) or "Pérdida" (Loss) generated by the spread between the official and parallel exchange rates.

### 3. Crowdsourced Price Guide (Comunidad)
* Every time a user adds a priced item to their personal calculator, it is anonymously (first-name only) pushed to the global `productos_comunidad` Firebase database.
* The "Explorar" tab groups these products and ranks the supermarkets by price, highlighting the cheapest option with a trophy (🏆).

### 4. Cloud Sync & Multi-Lists
* Google Authentication via Firebase.
* Users can save multiple lists to the cloud.
* **"Cargar a Calculadora" (Resume):** Users can push a saved list back into the live calculator, update prices, and hit "Save" to overwrite the existing list or create a new copy.

### 5. Native App Feel (PWA & UX)
* **Soft UI & Dark Mode:** Uses CSS variables to automatically switch between Light and Dark mode based on the user's OS settings.
* **Aisle Navigation:** Replaced infinite scrolling with categorical "Aisles" (Pasillos) to quickly find products.
* **Pull-to-Refresh:** Custom touch-event logic to mimic native pull-to-refresh functionality without moving the screen layout.

## 🛠️ Tech Stack & Architecture

This project is built using vanilla web technologies to keep it lightweight, fast, and easy to host on GitHub Pages without a build step (no Node.js, Webpack, or React required).

* **Frontend:** HTML5, CSS3, Vanilla JavaScript (ES6 Modules).
* **Backend / Database:** Firebase Firestore (NoSQL).
* **Authentication:** Firebase Auth (Google Provider).
* **Hosting:** GitHub Pages.
* **External APIs:** [PyDolarVenezuela API](https://pydolarvenezuela-api.vercel.app/)

## 📁 File Structure

For any developer or AI continuing this project, the separation of concerns is strictly maintained across three main files:

| File | Purpose |
|------|---------|
| `index.html` | The main view and application logic. Contains the DOM structure, modal overlays, Firebase initialization, state management (the `carrito` array), and CRUD operations. |
| `style.css` | All styling. Uses `:root` CSS variables for theming. Implements a fixed bottom-nav, flexbox layout, and prevents horizontal scrolling. |
| `catalogo.js` | The static database mapping. Contains the `catalogoPorCategorias` object (Aisles -> Products), `supermercadosFijos` array, and `marcasOficiales` array. |
| `manifest.json` | PWA manifest defining the app name, standalone display mode, theme colors, and homescreen icons. |

## 🗄️ Database Schema (Firestore)

The Firebase Firestore database is structured into two main collections:

### 1. `productos_comunidad` (Global Collection)
Stores crowd-sourced prices. Every document represents a single price report.
```javascript
{
  producto: "Arroz (Mary)",      // String
  precioUSD: 1.20,               // Number
  supermercado: "Gama",          // String
  unidad: "und",                 // String (und, kg, g, L)
  fecha: timestamp,              // Firestore Timestamp
  reportadoPor: "Juan"           // String (User's first name or "Anónimo")
}

### 2. usuarios/{uid}/historial (Private Sub-collections)
Stores the user's saved shopping lists.
{
  nombre: "Mercado Quincena",    // String
  fecha: "12/5/2026",            // String (Locale Date)
  timestamp: timestamp,          // Firestore Timestamp
  items_array: [                 // Array of Objects (The live state of the cart)
    { id: 16234..., name: "Arroz", price: 1.2, qty: 2, unit: "und", store: "Gama", isCommunity: true }
  ],
  html: "<li>...</li>",          // String (Legacy visual backup)
  total: 2.40,                   // Number (Total USD)
  tasaBCV_guardada: 36.50,       // Number
  tasaBinance_guardada: 41.50,   // Number
  totalVES_guardado: 87.60,      // Number
  totalUSDT_guardado: 2.11,      // Number
  ahorro_guardado: 0.29          // Number
}

🤖 Instructions for AI Models
If you are an AI assisting with this project, keep the following in mind:

State Management: The live cart is maintained in the carrito array in index.html. Any changes to the UI list must reflect changes in this array. Always call window.renderCarrito() after modifying the array.

Local Storage: The app uses localStorage to persist the active draft (mercadoVE_carrito_v2) and currently loaded list IDs (mercadoVE_lista_id) across page reloads.

Vanilla JS: Do not suggest adding React, Vue, or build tools (NPM). All Firebase imports must use the gstatic CDN URLs (e.g., https://www.gstatic.com/firebasejs/10.8.1/...).

Modo Planificación: A price of 0 means the item is "Pending" (Pendiente ⏳). Do not block items from being added to the cart if their price is 0.

🚀 Setup for Local Development
Clone the repository.

Serve the directory using any local web server (e.g., VS Code Live Server, Python http.server).

Note: Running directly from the file:// protocol will break ES6 Modules and Firebase Auth.

Firebase Security: The Firebase config keys in index.html are restricted to the production GitHub Pages domain and localhost. If deploying to a new domain, you must add the new domain to the Authorized Domains in the Firebase Authentication settings.
